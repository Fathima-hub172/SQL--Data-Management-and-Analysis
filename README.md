# SQL Data Management & Analysis 📂

## Overview
This project demonstrates the use of **SQL** to manage and analyze multiple datasets across different domains.  
It includes tables for students, employees, courses, products, and sales, showcasing skills in **data modeling, query design, joins, views, stored procedures, triggers, and aggregate functions**.

---

## Dataset Coverage
- **Students Table** → StudentID, Name, Department, GPA, Email  
- **Courses Table** → CourseID, CourseName, Credits, Instructor  
- **Employees Table** → EmployeeID, Name, Department, Salary, HireDate  
- **Products Table** → ProductID, ProductName, Price, Category  
- **Sales Table** → SaleID, ProductID, Quantity, SaleDate, TotalAmount  

---

## Features
- Table creation and normalization for different domains  
- Insert, update, and delete operations with constraints  
- Queries using `JOIN`, `GROUP BY`, `HAVING`, `ORDER BY`, `LIMIT`  
- Advanced SQL: Views, Stored Procedures, Triggers, User‑Defined Functions  
- Data cleaning examples (handling NULLs, duplicates, inconsistent values)  

---

## Outcomes
- **Structured Database**: Organized multiple datasets into relational tables.  
- **Query Insights**:  
  - Top 3 highest‑priced products  
  - Department‑wise employee counts and salaries  
  - Student GPA analysis by course and department  
  - Sales trends and revenue contribution by product category  
- **Reusable SQL Assets**: Created views, procedures, and triggers for automation.  
- **Portfolio Value**: Demonstrates ability to handle diverse datasets and apply SQL for real‑world scenarios.  

---

## Tech Stack
- SQL (MySQL / PostgreSQL / SQL Server)
